# 创作中心初步设计文档

## 系统描述
多Agent协调驱动的宠物自媒体创作系统

## Agent定义
1. Lead Agent：主链路流程，负责实际与用户进行交互，并协调使用其他Agent进行任务执行
    - Plan: 负责根据用户的需求、当前实时的状态，创建或更新当前的执行计划，维护Todo列表
    - Chat: 通过流式输出与用户进行交流
    - Simple Execute：对于简单的任务，直接调用工具来完成任务
    - Complex Dispatch：对于复杂的任务，根据任务的复杂度和依赖关系，调度给其他Agent来完成
    - Ask User Question: 对于任何不清晰的需求，询问用户更多细节以完善当前上下文
2. Hot Video Analyze Agent：爆款分析Agent，当用户指定了某个爆款视频时，通过多模态大模型对视频进行拆解和逐帧解析
    - Hot Reason: 分析视频的热门原因，包括但不限于视频内容、视频风格、视频标签等
    - Copy Advice: 提供复刻、跟拍视频的建议
3. Script Agent: 脚本Agent，负责根据要求生成视频脚本。
    - Script Pre Discussion: 与用户进行充分的脚本创作前的讨论，直到用户和Agent均认为没有其他问题
    - Script Generation: 根据讨论的结果，生成视频脚本
    - Script Refinement: 对生成的脚本进行优化和调整，确保脚本符合用户的需求
4. Shot Agent: 分镜Agent，负责根据脚本生成分镜
    - Shot Pre Discussion: 与用户进行充分的分镜创作前的讨论，直到用户和Agent均认为没有其他问题
    - Shot Generation: 根据脚本，生成视频的分镜
    - Shot Refinement: 对生成的分镜进行优化和调整，确保分镜符合用户的需求
    - Shot Draft: 分镜草稿图，根据调整好的分镜生成分镜的草稿图（需要图像生成模型）
5. 支持根据实际需要后续扩展Agent

## 前端界面
前端页面采用左右结构进行布局：
### 左侧
- 初始为项目列表，支持增删项目，仅在项目被选中时才能与右侧Agent进行交互
- 选中项目后进入项目详情，存储Agent生成的分析结果、脚本、分镜等产出内容，用户可进行查看和编辑
### 右侧
- 右侧为Agent交互界面，采用成熟的LLM聊天窗口界面，支持用户与Agent进行实时交互，并支持发送文件、图片、视频等多媒体内容

## 系统规划
基于现在backend中现有的Agent架构进行扩展，整体遵循以下原则
1. Lead Agent作为主入口，负责判断是否需要调用其他Agent
2. 前后端交互采用SSE形式，流式输出执行结果
3. 对于中间任何操作（调用子Agent、调用工具），均需要支持在前端进行展示
4. 整体上下文以项目维度作区分，所有上下文需要持久化在sqlite中

## 工具实现
配置页面新增工具配置tab，并在后端实现以下built-in的工具
1. Todo Write: 写入、更新、删除Todo列表
2. Shell: 命令行工具，支持沙箱、白名单执行
3. Video Analyze: 视频分析工具，支持在配置页面指定使用的多模态大模型
4. File Reader： 文件读取工具，支持读取本地文件
5. File Editor：文件编辑工具，支持进行文件的创建、编辑、删除
